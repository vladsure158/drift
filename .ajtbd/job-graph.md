## Core Job 1: "Переключиться на проект и мгновенно продолжить работу"

**Когда** я переключаюсь между 3-7 проектами в день, **хочу** мгновенно видеть где остановился и что осталось

### 1.1 Вспомнить какие проекты у меня вообще есть

- когда
  - контекст: сажусь работать утром или после перерыва, открываю терминал/IDE. Проектов 10-30 в разных папках
  - триггер: "так, чем я сегодня займусь?" — открываю Finder/терминал и вижу десятки папок
  - эмоции в точке А: лёгкая тревога, ощущение хаоса, "я точно что-то забыл"

- хочу получить ожидаемый результат: полный список моих активных проектов с минимальным контекстом
  - критерии успеха: вижу все проекты за < 3 секунды; неактивные не мешают; понятно что "живое"

- Проблемы: Проекты разбросаны по файловой системе, нет единого реестра. Приходится вспоминать по памяти или листать папки.
- Сила проблемы: 8/10

### 1.2 Выбрать каким проектом заняться прямо сейчас

- когда
  - контекст: вижу список проектов, нужно решить на чём сфокусироваться
  - триггер: "что сейчас приоритетнее?" — мысленный перебор проектов
  - эмоции в точке А: нерешительность, decision fatigue, прокрастинация

- хочу получить ожидаемый результат: быстро принять решение на основе прогресса, срочности и мотивации
  - критерии успеха: решение принято за < 30 секунд; не чувствую что упускаю что-то важное

- Проблемы: Нет наглядного сравнения проектов по приоритету/прогрессу. Выбираю что проще, а не что важнее.
- Сила проблемы: 6/10

### 1.3 Открыть рабочее окружение нужного проекта

- когда
  - контекст: выбрал проект, нужно перейти в директорию, открыть IDE
  - триггер: "ок, работаю над landing-saas" — ищу папку
  - эмоции в точке А: раздражение если не помню точный путь

- хочу получить ожидаемый результат: за одно действие оказаться в директории проекта
  - критерии успеха: одна команда; не нужно вспоминать путь

- Проблемы: Надо помнить путь к директории. Потеря 30-60 секунд на поиск.
- Сила проблемы: 5/10

### 1.4 Восстановить контекст — что было сделано в прошлый раз

- когда
  - контекст: открыл проект, смотрю на код. Последний раз работал 2 дня — неделю назад
  - триггер: смотрю на файлы, пытаюсь вспомнить "а, точно, я же тут..."
  - эмоции в точке А: фрустрация, "опять 10 минут на вспоминание", потерянное время

- хочу получить ожидаемый результат: за 10-15 секунд понять что сделано, на чём остановился, какие проблемы были
  - критерии успеха: не нужно читать git log и код; вижу заметки "прошлый я" оставил "будущему мне"

- Проблемы: Git log показывает только коммиты, не контекст мышления. Заметки разбросаны. Часто контекст нигде не зафиксирован — 5-15 мин на восстановление КАЖДЫЙ раз.
- Сила проблемы: 9/10 ← САМАЯ СИЛЬНАЯ ПРОБЛЕМА

### 1.5 Понять что нужно делать дальше

- когда
  - контекст: восстановил контекст, нужен следующий шаг
  - триггер: "окей, вспомнил, теперь что делать?"
  - эмоции в точке А: неопределённость, путаница между проектами

- хочу получить ожидаемый результат: чёткий список того, что осталось — цели, задачи, приоритеты
  - критерии успеха: вижу конкретные next steps; не путаю задачи между проектами

- Проблемы: TODO-листы разбросаны (README, Notion, стикеры, голова). Часто TODO вообще нет.
- Сила проблемы: 7/10

### 1.6 Начать работу — запустить Claude/Cursor и кодить

- когда
  - контекст: контекст восстановлен, знаю что делать
  - триггер: "поехали" — пишу промт
  - эмоции в точке А: облегчение, flow

- хочу получить ожидаемый результат: продуктивно работать, инструмент не мешает
  - критерии успеха: инструмент управления фоново фиксирует прогресс

- Проблемы: минимальные на этом шаге
- Сила проблемы: 2/10

### 1.7 Зафиксировать прогресс перед завершением сессии

- когда
  - контекст: поработал 30 мин — 3 часа. Пора переключаться
  - триггер: "всё, хватит на сегодня по этому проекту"
  - эмоции в точке А: спешка, лень ("а, потом запишу" → никогда)

- хочу получить ожидаемый результат: за 10-15 сек зафиксировать что сделал, на чём остановился, что дальше
  - критерии успеха: быстро (< 15 сек); не выбивает из ритма

- Проблемы: Нет привычки и инструмента. Git commit фиксирует код, но не контекст. Если > 30 сек — не делаю.
- Сила проблемы: 8/10

## Core Job 2: "Зарегистрировать новый вайб-проект мгновенно"

**Когда** запускаю очередной вайб-проект с Claude/Cursor, **хочу** за секунду зарегистрировать его в системе

### 2.1 Получить идею и решить "делаю!"

- когда
  - контекст: увидел идею (Twitter, Telegram, разговор). Азарт: "это реально сделать за вечер с AI!"
  - триггер: конкретная идея + ощущение "реально быстро"
  - эмоции в точке А: возбуждение, "хочу начать ПРЯМО СЕЙЧАС"

- хочу получить ожидаемый результат: максимально быстро от идеи к коду
  - критерии успеха: от идеи до первой строки < 2 минут

- Проблемы: Любое трение убивает мотивацию
- Сила проблемы: 7/10

### 2.2 Создать проект (директория, git, scaffold)

- когда
  - контекст: mkdir, npx create-next-app, git init
  - триггер: решение принято, руки на клавиатуре
  - эмоции в точке А: нетерпение

- хочу получить ожидаемый результат: рабочая директория готова к вайб-кодингу
  - критерии успеха: < 60 секунд; git инициализирован

- Проблемы: Уже оптимизировано фреймворками
- Сила проблемы: 3/10

### 2.3 Зарегистрировать проект в системе учёта

- когда
  - контекст: проект создан, нужно добавить в "реестр". Реестра нет — проект просто папка
  - триггер: "надо бы записать что я это начал..." (если возникает)
  - эмоции в точке А: "не хочу бюрократию, хочу кодить"

- хочу получить ожидаемый результат: проект автоматически попадает в систему учёта с базовыми метаданными
  - критерии успеха: 0-5 секунд; не нужно заполнять формы; авто-определяет стек/имя/путь

- Проблемы: Этого шага просто НЕТ — нет инструмента. Через неделю забываю что проект существует.
- Сила проблемы: 8/10

### 2.4 Записать идею / цель проекта

- когда
  - контекст: проект создан, хочу зафиксировать ЗАЧЕМ одним предложением
  - триггер: "через неделю я не вспомню зачем эта папка"
  - эмоции в точке А: "ну ладно, одно предложение запишу"

- хочу получить ожидаемый результат: краткое описание зафиксировано
  - критерии успеха: < 10 сек, необязательно (можно дописать потом)

- Проблемы: README дефолтный. Через месяц: "landing-saas? какой SaaS?"
- Сила проблемы: 6/10

### 2.5 Начать кодить (и забыть про менеджмент)

- когда
  - контекст: всё готово, открываю Claude Code
  - триггер: первый промт
  - эмоции в точке А: радость, flow

- хочу получить ожидаемый результат: кодить без отвлечений
  - критерии успеха: инструмент учёта работает фоново

- Проблемы: нет — целевое состояние
- Сила проблемы: 1/10

## Core Job 3: "Понять какие проекты продолжать, а какие бросить"

**Когда** у меня 20+ проектов, **хочу** понять какие продолжать, а какие бросить

### 3.1 Собрать обзор всех проектов в одном месте

- когда
  - контекст: раз в 1-2 недели сажусь "навести порядок"
  - триггер: "я уже не помню сколько у меня проектов" или перегруженность
  - эмоции в точке А: перегруженность, тревога, "я тону"

- хочу получить ожидаемый результат: полный список ВСЕХ проектов с статусом, прогрессом, свежестью
  - критерии успеха: ничего не пропущено; актуально; вижу картину за 30 секунд

- Проблемы: Вручную по папкам — 20-40 мин. Часть проектов забыта.
- Сила проблемы: 9/10

### 3.2 Оценить прогресс каждого проекта

- когда
  - контекст: вижу список, нужно понять где каждый проект
  - триггер: смотрю на название и пытаюсь оценить "что там?"
  - эмоции в точке А: неуверенность

- хочу получить ожидаемый результат: для каждого проекта — объективная оценка прогресса
  - критерии успеха: не надо открывать каждый проект; основано на фактах, а не ощущениях

- Проблемы: Нет формализованных целей, невозможно измерить %. На 20 проектов — час+.
- Сила проблемы: 7/10

### 3.3 Оценить "свежесть"

- когда
  - контекст: часть проектов не трогал неделями
  - триггер: "этот проект 3 недели не трогал — значит не нужен?"
  - эмоции в точке А: вина, неопределённость

- хочу получить ожидаемый результат: видеть когда работал + тренд (активен / затухает / заброшен)
  - критерии успеха: автоматически из git; визуально понятно

- Проблемы: git log по каждому проекту отдельно. Нет агрегированного вида.
- Сила проблемы: 6/10

### 3.4 Принять решение: продолжить / пауза / бросить

- когда
  - контекст: вижу проект, понимаю состояние. Нужно решить
  - триггер: "у меня 20 проектов, могу вести 5 — какие?"
  - эмоции в точке А: FOMO, sunk cost fallacy, "жалко бросать"

- хочу получить ожидаемый результат: решение зафиксировано: active / paused / abandoned / done
  - критерии успеха: решение осознанное; зафиксировано; можно вернуться к paused

- Проблемы: Нет формального "архивирования". Проекты не "бросаются" — "забываются". Нет разницы между "пауза" и "забыл".
- Сила проблемы: 7/10

### 3.5 Расставить приоритеты

- когда
  - контекст: 3-7 активных проектов, нужен порядок
  - триггер: "какой первый?"
  - эмоции в точке А: облегчение + мотивация

- хочу получить ожидаемый результат: ранжированный список с next step для каждого
  - критерии успеха: порядок зафиксирован; не принимаю заново завтра

- Проблемы: Приоритеты в голове, завтра снова с нуля
- Сила проблемы: 5/10

### 3.6 Зафиксировать результаты ревью

- когда
  - контекст: провёл ревью, принял решения
  - триггер: "записать бы..."
  - эмоции в точке А: усталость, "и так запомню" (не запомню)

- хочу получить ожидаемый результат: статусы и решения сохранены
  - критерии успеха: следующее ревью отталкивается от предыдущего

- Проблемы: Нет единого места. Каждое ревью с нуля.
- Сила проблемы: 7/10

## Критическая последовательность работ (КПР)

```
Нет реестра (2.3, 8) → Не могу собрать обзор (3.1, 9) → Не фиксирую прогресс (1.7, 8)
                                                            ↓
                                              Не могу восстановить контекст (1.4, 9)
```

Все четыре TOP-проблемы решаются одним ядром: единый реестр + быстрые заметки + auto-discovery.
